Artifact 2: Machine Learning vs. Deep Learning

I compared traditional machine learning and deep learning through two real-world problem types: email spam detection and image recognition.

View the Final Report View or Download Portfolio Documentation View Value Proposition Back to Portfolio

Project Overview

This project examines how the structure of a problem and its data affects the choice between traditional machine learning and deep learning. I used email spam detection to study a structured binary-classification problem and image recognition to study a problem based on raw, high-dimensional visual data.

Objective

My objective was to explain why logistic regression is suitable for a traditional machine-learning classification problem and why a convolutional neural network is suitable for image recognition. I also compared why the alternative approach would be less practical in each case.

Value Proposition

This artifact gives a reviewer a concise, evidence-based view of how I reason about model selection. It demonstrates how problem structure, feature representation, interpretability, data volume, computational cost, and real-world variation guide the choice between traditional machine learning and deep learning.

Unique Value

The side-by-side case studies make the technical distinction concrete: logistic regression scores engineered features for a binary outcome, while a CNN learns layered visual features directly from raw pixels. The artifact also separates verified product capabilities from illustrative algorithm choices, demonstrating responsible technical communication.

Traditional Machine Learning Example

Email Spam Detection with Logistic Regression

Email spam detection is a binary-classification problem because each message must be classified as spam or not spam. Logistic regression can use structured numerical inputs such as word frequencies, links, sender characteristics, and message length to estimate the probability that a message belongs to the spam class.

This approach is suitable because it is efficient, produces class probabilities, and offers relative interpretability through its feature coefficients. Gmail provides a real-world context for machine-learning-based spam filtering, but my report evaluates logistic regression as an appropriate traditional method; it does not claim that Gmail currently uses logistic regression in production.

Deep-Learning Example

Image Recognition with a Convolutional Neural Network

Image recognition begins with raw pixel values rather than predefined features. A convolutional neural network can learn visual features in stages: early layers identify patterns such as edges and gradients, middle layers combine them into shapes and textures, and deeper layers recognize complete objects or faces.

Google Photos provides a real-world context through AI-supported photo organization, content search, and face grouping. My report uses CNN and FaceNet research to explain why deep learning is suitable for image-recognition problems; it does not claim to identify the exact model or internal architecture used by Google Photos.

Logistic Regression and CNN Comparison

▤

Logistic Regression

Best suited here to structured inputs and two-class prediction when useful features can be selected before training. It is comparatively efficient and interpretable.

◉

Convolutional Neural Network

Best suited here to raw visual data when useful features must be learned automatically across multiple levels of abstraction.

Neither approach is universally better. Traditional machine learning is a strong fit when features are already understood and can be structured. Deep learning becomes more useful when the input is unstructured and the system must discover complex features from examples.

Real-World Applications

The two case studies connect course concepts to familiar applications. Spam filtering shows how classification can separate unwanted email from legitimate messages. Image recognition shows how visual systems can support photo organization, search, face grouping, recognition, verification, and clustering.

  • Connecting technical concepts to real-world applications

Challenges and Solutions

Choosing Comparable Examples

The two examples needed to show a meaningful difference between traditional and deep learning. I addressed this by comparing a structured binary-classification problem with a raw visual-data problem and focusing on how each approach handles features.

Keeping Product Claims Accurate

A real-world product can demonstrate an application without confirming its complete internal model. I separated documented product capabilities from the algorithms evaluated in the report and stated the limits of those comparisons clearly.

Explaining Technical Differences Clearly

The comparison could become too abstract if it focused only on terminology. I addressed this by connecting feature engineering, probability outputs, layered visual learning, interpretability, data requirements, and computational complexity to the two case studies.
